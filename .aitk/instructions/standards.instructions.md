# Language & Coding Standards

## 1. Language Policy

- **Codebase:** All code (variable names, comments, docstrings, commit messages) MUST be in **English**.
- **Documentation:** All documentation (README, CONTRIBUTING, docs/) MUST be in **English**.
- **Reasoning:** This is an international open-source project. English ensures accessibility for global contributors and judges.

## 2. Python Standards

- **Style:** Follow PEP 8.
- **Type Hinting:** Use strict type hints for all function signatures.
- **Docstrings:** Use Google-style docstrings for all public functions and classes.

## 3. Gradio / UI

- **Labels:** UI labels and user-facing text should be in English.
- **Responsiveness:** Avoid hardcoded dimensions (width/height) unless strictly necessary. Rely on container layouts.
